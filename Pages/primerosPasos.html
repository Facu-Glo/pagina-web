Primeros pasos

Linux es un sistema operativo que, al igual que otros sistemas como Windows o macOS, se puede utilizar sin necesidad de interactuar directamente con la terminal. Muchas aplicaciones gráficas están disponibles, y la mayoría de las tareas cotidianas se pueden realizar usando interfaces gráficas, como el explorador de archivos o programas específicos.

¿Qué es la terminal?

Es un intérprete de comandos u órdenes (conocido como terminal, consola o shell) que hace de interfaz entre el usuario y el propio sistema operativo. Es decir, mediante la terminal podemos acceder al sistema operativo sin utilizar la interfaz gráfica y realizar todo tipo de tareas en modo texto.

Linea de comandos

Los comandos de Linux te permiten controlar tu sistema desde la interfaz de línea de comandos (CLI) en lugar de utilizar tu ratón o trackpad. Son instrucciones de texto introducidas en el terminal para indicarle a tu sistema exactamente qué hacer.

Los comandos que ingreses en el terminal de Linux distinguen entre mayúsculas y minúsculas y siguen una sintaxis como “comando -opciones argumentos“. Puedes combinarlos para tareas complejas mediante pipelines y redirecciones.

Comandos Básicos de Linux

  • ls - Listar contenido del directorio:

    • Lista el contenido de un directorio, mostrando todos los archivos y subdirectorios contenidos en su interior.

    Sin opciones o argumentos, ls mostrará el contenido del directorio de trabajo actual. Puedes pasar un nombre de ruta para listar archivos y carpetas en esa ubicación en su lugar.

    Sintaxis

    ls [opciones] [directorio]

    Las opciones pueden ser:

    • -a - Muestra todos los archivos, incluidos los ocultos.
    • -l - Muestra una lista detallada de archivos y directorios.
    • -R - Lista de forma recursiva todo el contenido de los subdirectorios, descendiendo indefinidamente en carpetas secundarias.
    • -S - Ordena los resultados por tamaño de archivo, comenzando por el más grande.
  • cd - Cambiar de directorio:

    • El comando cd se utiliza para navegar entre directorios. Te permite moverte desde el directorio de trabajo actual a una nueva ubicación en el sistema de archivos.

    Cuando ejecutas el comando cd por sí mismo, te llevará al directorio de inicio. También puedes pasar una ruta específica para cambiar a ella. Por ejemplo:

    • cd /usr/local – Cambia al directorio /usr/local
    • cd .. – Te mueve un nivel atrás hacia el directorio principal
  • mkdir - Crear un nuevo directorio:

    • El comando mkdir te permite crear una nueva carpeta. Simplemente, específicas el nombre del directorio a crear.

    Sintaxis

    mkdir [opción] [directorio]

    Algunas opciones útiles de mkdir:

    • -p Crea directorios principales de forma recursiva según sea necesario.
    • -v Muestra un resultado detallado que indica los directorios creados.
  • cp- Copiar archivos y directorios

    • El comando cp copia archivos o directorios de una ubicación a otra. Requiere pasar una ruta de origen y un destino.

    Sintaxis

    cp [opciones] [origen] [destino]

    Opciones útiles:

    • -r Copia directorios de forma recursiva, descendiendo a subdirectorios para copiar su contenido también.
    • -v Muestra un resultado detallado que indica los detalles de cada archivo mientras se copia. Útil para confirmar exactamente lo que se copió.
    • –i Copia un fichero o directorio y pregunta antes de sobrescribir si se diese el caso.
  • mv - Mover o renombrar archivos y directorios

    • El comando mv se utiliza para mover archivos o directorios a una ubicación diferente o renombrarlos. A diferencia del comando anterior, los archivos de la ruta de origen se eliminan después de haber sido movidos al destino.

    También puedes usar el comando mv para renombrar archivos, ya que solo necesitas cambiar las rutas de origen y destino al antiguo y nuevo nombre.

    Sintaxis

    mv [opciones] [origen] [destino]
  • rm - Eliminar archivos y directorios

    • El comando rm elimina archivos y directorios. Úsalo con precaución porque los archivos y directorios eliminados no se pueden recuperar.

    Sintaxis

    rm [opciones] [nombre]

    Opciones útiles

    • -r – Elimina directorios de forma recursiva, incluyendo todos los contenidos dentro de ellos. Necesario al eliminar directorios.
    • -f – Forzar la eliminación y suprimir todas las solicitudes de confirmación.
    • -i – Solicitar confirmación antes de eliminar cada archivo o directorio. Proporciona seguridad contra borrado accidental.

    Advertencia: El comando rm borra permanentemente archivos y carpetas, así que úsalo con extrema precaución. Si se usa con privilegios de sudo, también podrías eliminar por completo el directorio raíz y Linux dejará de funcionar después de reiniciar tu computadora.

  • touch - Crear un nuevo archivo vacio

    • El comando touch se utiliza para crear instantáneamente un nuevo archivo vacío. Esto es útil cuando necesitas un archivo vacío para llenarlo con datos más tarde.

    Sintaxis

    touch [opciones] [nombredearchivo]

    Opciones

    • -c - No crea el archivo si ya existe. Evita sobrescribir archivos existentes accidentalmente.
    • -m - En lugar de crear un nuevo archivo, actualiza la marca de tiempo en un archivo existente. Esto se puede usar para cambiar la hora de modificación
  • cat

    • Se utiliza para mostrar el contenido de un archivo, concatenar múltiples archivos o crear nuevos archivos. Es uno de los comandos más básicos y útiles cuando se trabaja con la línea de comandos.

    Sintaxis

    cat [opciones] [archivo(s)]

    Funciones:

    Mostrar el contenido de un archivo

    cat archivo.txt

    Concatenar múltiples archivos: Muestra el contenido de varios archivos uno después de otro. Esto es útil para ver el contenido de varios archivos sin abrir cada uno por separado.

    cat archivo1.txt archivo2.txt

    Crear un nuevo archivo: Puedes crear un archivo y escribir contenido en él desde la terminal. Usa Ctrl+D para finalizar la entrada.

    cat > nuevoarchivo.txt

    Opciones

    • -n - Muestra el contenido del archivo con números de línea.
    • -b - Similar a -n, pero no numera las líneas en blanco.

Lo importante de un sistema: las aplicaciones

Un sistema operativo, por muy poderoso que sea, no sirve de mucho sin aplicaciones que nos permitan realizar nuestras tareas del día a día. Ya sea para navegar por internet, editar documentos, jugar, o programar, las aplicaciones son la esencia de cualquier experiencia informática. Por eso, contar con un buen sistema para gestionar nuestras aplicaciones es clave.

Ahí es donde entran en juego los gestores de paquetes.

¿Qué es un gestor de paquetes?

Un gestor de paquetes es una herramienta que facilita la instalación, actualización, configuración y eliminación de aplicaciones en tu sistema. En lugar de descargar programas desde sitios web como en otros sistemas operativos, en Linux utilizamos gestores de paquetes para acceder a repositorios oficiales, que son colecciones seguras y organizadas de software. Esto no solo te ahorra tiempo, sino que garantiza que el software sea confiable y esté optimizado para tu sistema.

Mi recomendación personal

En mi experiencia, instalar aplicaciones desde la terminal usando un gestor de paquetes es rápido, limpio y eficiente. Prefiero este método porque me da más control y me asegura que siempre estoy obteniendo versiones actualizadas directamente desde los repositorios oficiales. Además, no necesitas depender de tiendas de aplicaciones gráficas que pueden ser más lentas o complicadas.

Si es nuevo, no se preocupe. Con unos pocos comandos básicos puede instalar, actualizar y eliminar programas sin complicaciones.

Es importante tener en cuenta que cada distribución de Linux utiliza un gestor de paquetes diferente, dependiendo de cómo está diseñada. Algunos ejemplos comunes son:

  • Arch Linux y derivados: Usan pacman. Este gestor es conocido por su rapidez y simplicidad. Los comandos son cortos y fáciles de recordar.
  • Debian y derivados (como Ubuntu y Linux Mint): Utilizan apt, que es muy popular y ampliamente documentado.
  • Fedora y Red Hat: Usan dnf o su predecesor, yum. Estos gestores son potentes y manejan dependencias de forma eficiente.

Aunque los comandos cambian de un gestor a otro, la idea básica es la misma: instalar, actualizar y eliminar aplicaciones de manera sencilla y confiable.

Instalacion de aplicaciones

En la mayoría de las distribuciones de Linux, puedes instalar aplicaciones desde la terminal utilizando el comando sudo, seguido del gestor de paquetes específico de tu distribución, una instrucción como install, y el nombre del paquete que deseas instalar. Por ejemplo: 

                    
                        sudo [gestor-de-paquetes] [instrucción] [nombre-del-paquete]

Por ejemplo, en Ubuntu, el gestor de paquetes es apt, así que para instalar el navegador Firefox: 

                    
                        sudo apt install firefox

Las instrucciones básicas que se utilizan comúnmente con los gestores de paquetes son:

  • install
  • remove
  • update
  • search

En Arch Linux, las instrucciones básicas son un poco diferentes y se usan los siguientes comandos:

  • -S: para instalar un paquete. Ejemplo: sudo pacman -S firefox
  • -R: para eliminar un paquete. Ejemplo: sudo pacman -R firefox
  • -Ss: para buscar un paquete en los repositorios. Ejemplo: pacman -Ss firefox
  • -Syu: para actualizar todos los paquetes instalados en el sistema. Ejemplo: sudo pacman -Syu

Recuerda siempre verificar el gestor de paquetes específico de tu distribución, ya que las instrucciones pueden variar.

Dominar los comandos básicos de Linux es fundamental para quienes desean aprovechar al máximo este sistema operativo. Aunque las interfaces gráficas son útiles, trabajar con la terminal ofrece un control más preciso y rápido sobre el sistema, facilitando tareas de administración, automatización y personalización. Los comandos mencionados son solo el principio, y con el tiempo descubrirás muchos otros que te permitirán realizar tareas aún más complejas.

Si estás comenzando con Linux, te animo a que practiques estos comandos de forma regular para familiarizarte con ellos. La terminal puede parecer intimidante al principio, pero con paciencia y práctica, te volverás más ágil y eficiente.
